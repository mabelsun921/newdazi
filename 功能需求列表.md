# 功能需求列表

## 1. 活动广场模块

### 1.1 核心功能
- **活动发布**：用户可以发布活动，填写结构化信息
- **活动浏览**：用户可以浏览活动列表
- **申请参加**：用户可以申请参加活动，等待发起者确认
- **活动搜索**：用户可以通过文字搜索活动，大模型进行理解并匹配
- **活动推荐**：根据用户结构化信息、爱好、过去参与的活动等个性化内容给活动评分并推荐

### 1.2 活动发布信息
- 主题
- 活动内容简介
- 地点
- 预算
- 人数范围
- 对参与者的要求
- 是否需要发起者通过确认才能参与
- 活动押金（虚拟币）

### 1.3 附加功能
- **AI帮写**：用户可以输入活动链接，AI帮助拆解并填写活动信息
- **AI活动提示**：用户可以输入自然语言描述活动，AI自动生成活动标题、分类、标签、简介等信息，当大模型API不可用时使用本地规则引擎兜底

### 1.4 交互功能
- 点击活动中用户头像进入该用户主页
- 通过主页的聊天按钮进行聊天

## 2. 灵魂匹配模块

### 2.1 核心功能
- **用户匹配**：根据注册用户的结构化信息和参与的活动风格等内容匹配好友
- **喜欢/不喜欢**：用户可以对匹配到的用户进行喜欢或不喜欢操作
- **匹配限制**：每天可以匹配三个用户

### 2.2 交互流程
- 喜欢：自动进入聊天对话（复用消息列表里的对话框），对话完毕返回到消息列表
- 不喜欢：滑走该用户

## 3. 消息列表模块

### 3.1 核心功能
- **对话管理**：展示与其他人的对话列表，类似微信对话
- **聊天功能**：可以与其他人沟通交流活动问题或其他聊天内容

### 3.2 交互功能
- 点击用户头像可以看到用户过去参与的活动、正在发起的活动、其他人对用户的评价

## 4. 个人中心模块

### 4.1 核心功能
- **个人信息**：展示用户的个人信息
- **参与活动**：展示用户参与的活动（已结束的、正在发起的、已经报名的）
- **评价信息**：展示其他人对用户的评价

## 5. 技术实现要点

### 5.1 后端技术
- Flask REST API
- MongoDB 数据库
- Gemini API 集成（AI功能）

### 5.2 前端技术
- React
- 响应式设计
- 黑色主题
- 底部导航

### 5.3 数据模型
- 用户表（users）
- 用户资料表（profiles）
- 活动表（activities）
- 活动申请表（activity_applications）
- 聊天表（chats）
- 消息表（messages）
- 评价表（reviews）

### 5.4 API 端点
- 用户认证相关
- 活动相关（发布、浏览、申请、搜索、推荐）
- 灵魂匹配相关
- 消息聊天相关
- 个人中心相关
