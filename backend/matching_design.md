# 匹配系统设计文档

## 1. 广场匹配权重设计

### 1.1 权重因素及权重值

| 权重因素 | 权重值 | 说明 |
|---------|-------|------|
| 用户-发起者匹配度 | 30% | 基于用户与活动发起者的个人信息匹配程度 |
| 地域权重 | 25% | 基于用户与活动地点的距离远近 |
| 兴趣权重 | 20% | 基于用户兴趣标签与活动标签的匹配度 |
| 过往参与活动权重 | 15% | 基于用户过往参与类似活动的频率和评价 |
| 活动标签权重 | 5% | 基于活动标签的热门程度和相关性 |
| 发起者评分 | 5% | 基于活动发起者的信誉评分 |

### 1.2 评分计算方法

活动评分 = (用户-发起者匹配度 × 0.3) + (地域权重 × 0.25) + (兴趣权重 × 0.2) + (过往参与活动权重 × 0.15) + (活动标签权重 × 0.05) + (发起者评分 × 0.05)

### 1.3 评分等级

| 评分范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | S | 非常适合 |
| 80-89 | A | 很适合 |
| 70-79 | B | 适合 |
| 60-69 | C | 一般 |
| 0-59 | D | 不适合 |

## 2. 灵魂匹配权重设计

### 2.1 权重因素及权重值

| 权重因素 | 权重值 | 说明 |
|---------|-------|------|
| 性格匹配度 | 35% | 基于MBTI、性格描述等的匹配程度 |
| 兴趣爱好匹配度 | 25% | 基于兴趣标签、爱好的匹配程度 |
| 地域匹配度 | 15% | 基于地理位置的匹配程度 |
| 生活方式匹配度 | 15% | 基于生活习惯、作息时间等的匹配程度 |
| 价值观匹配度 | 10% | 基于个人价值观、人生目标等的匹配程度 |

### 2.2 评分计算方法

灵魂匹配评分 = (性格匹配度 × 0.35) + (兴趣爱好匹配度 × 0.25) + (地域匹配度 × 0.15) + (生活方式匹配度 × 0.15) + (价值观匹配度 × 0.1)

### 2.3 评分等级

| 评分范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 灵魂伴侣 | 非常匹配 |
| 80-89 | 知己 | 很匹配 |
| 70-79 | 好友 | 匹配 |
| 60-69 | 熟人 | 一般匹配 |
| 0-59 | 陌生人 | 不匹配 |

## 3. AI Prompt 设计

### 3.1 活动评分 Prompt

```
你是一个专业的活动匹配AI。你的任务是评估用户与活动的匹配度，给出0-100分的评分，并提供详细理由。

用户信息：
性别: {gender}
MBTI: {mbti}
职业: {occupation}
性格: {personality}
地点: {location}
兴趣标签: {ai_tags}
信誉分: {reputation}

活动信息：
标题: {title}
内容: {content}
分类: {category}
地点: {activity_location}
时间: {time}
发起者信息：
性别: {initiator_gender}
MBTI: {initiator_mbti}
职业: {initiator_occupation}
性格: {initiator_personality}
信誉分: {initiator_reputation}

请根据以下权重因素计算活动评分：
1. 用户-发起者匹配度 (30%)
2. 地域权重 (25%)
3. 兴趣权重 (20%)
4. 过往参与活动权重 (15%)
5. 活动标签权重 (5%)
6. 发起者评分 (5%)

请输出JSON格式，包含以下字段：
- score: 0-100的评分
- level: 评分等级 (S/A/B/C/D)
- reason: 详细评分理由
- breakdown: 各权重因素的具体得分
```

```
用户信息：
性别: {gender}
MBTI: {mbti}
职业: {occupation}
性格: {personality}
地点: {location}
兴趣标签: {ai_tags}

活动信息：
标题: {title}

时间: {time}

匹配分析：
- 评分: {score}
- 等级: {level}
- 详细理由: {reason}
- 各因素得分: {breakdown}

报告应该友好、专业，突出活动的吸引力和用户的契合度。
```

## 4. 实现建议

1. **权重计算实现**：
   - 使用Python实现权重计算逻辑
   - 地域权重可以使用地理距离计算
   - 兴趣匹配度可以使用标签交集计算
   - 性格匹配度可以基于MBTI兼容性矩阵

2. **AI集成**：
   - 使用Gemini API进行活动评分和报告生成
   - 为不同的匹配场景设计专门的prompt
   - 优化API调用，减少响应时间

3. **数据存储**：
   - 活动信息存储在activities集合
   - 用户信息存储在profiles集合
   - 匹配历史存储在match_history集合

4. **性能优化**：
   - 缓存常用的匹配结果
   - 批量处理匹配请求
   - 异步处理AI调用

## 5. 测试建议

1. **单元测试**：
   - 测试权重计算逻辑
   - 测试AI prompt效果
   - 测试API响应时间

2. **集成测试**：
   - 测试完整的匹配流程
   - 测试不同场景下的匹配结果
   - 测试边界情况

3. **用户测试**：
   - 收集用户对匹配结果的反馈
   - 优化匹配算法和权重
   - 改进AI报告质量

通过以上设计，系统可以为用户提供更加精准、个性化的活动推荐和灵魂匹配，提升用户体验和匹配成功率。
